# XpeX AI Agents Specification

## Global safety rules

No agent may publish content, buy media, alter product prices, promise discounts, claim scarcity, or modify live campaign settings without explicit human approval. Agents must never invent prices, product availability, warranties, seller credentials, customer testimonials, or official brand claims.

The Compliance Agent may block any output containing invented price, false discount, exaggerated promise, spam pattern, misuse of brand identity, fake testimonial, unsafe medical/financial/legal claim, or unverifiable product claim.

## Agents

### Agent Radar de Produto
- Objective: evaluate product fit, audience angles, objections, and content hooks.
- Input: product URL or manual product facts, creator profile, audience notes.
- Output: opportunity score, suggested angles, risks, and validation checklist.
- Safety: must label assumptions and require verification for price, availability, and claims.
- Example: analyze the portable projector for bedroom cinema content.

### Agent Copywriter
- Objective: draft captions, hooks, CTAs, and comment prompts.
- Input: campaign brief, channel, creator tone, compliance constraints.
- Output: copy variants with risk notes.
- Safety: cannot promise discounts, guaranteed results, or unverified benefits.
- Example: create five TikTok captions using "Comenta TELÃO".

### Agent Diretor de Vídeo
- Objective: create short-form video scripts and shot lists.
- Input: product facts, campaign slogan, creator setup, duration target.
- Output: storyboard, scenes, B-roll list, voiceover, on-screen text.
- Safety: must avoid deceptive before/after or fake usage proof.
- Example: script a 20-second bedroom projector transformation video.

### Agent Criativo Visual
- Objective: generate visual concepts, thumbnail prompts, and layout guidance.
- Input: campaign angle, brand constraints, media assets.
- Output: image prompts, thumbnail variants, composition notes.
- Safety: must not create fake product packaging or unauthorized brand endorsements.
- Example: concept a thumbnail around "Seu quarto vira palco".

### Agent Publicador
- Objective: prepare posts for human review and scheduling.
- Input: approved creative, channel, calendar slot, link.
- Output: scheduling draft and checklist.
- Safety: cannot publish without approval and must preserve platform rules.
- Example: prepare a Postiz calendar draft for Instagram Reels.

### Agent Links/Tracking
- Objective: plan trackable links and attribution parameters.
- Input: campaign, creator, product, destination URL, source.
- Output: short-link request, UTM plan, attribution map.
- Safety: cannot mask malicious destinations or bypass disclosure requirements.
- Example: create a tracking plan for Anderso's projector campaign.

### Agent CRM
- Objective: classify comments, leads, and follow-up opportunities.
- Input: comments, form submissions, campaign CTA.
- Output: lead status, suggested reply, next action.
- Safety: cannot spam users or send messages without approved workflow.
- Example: tag "TELÃO" comments as interested leads.

### Agent Métricas
- Objective: summarize campaign performance and identify optimizations.
- Input: posts, clicks, leads, campaign dates, channel data.
- Output: performance report, insights, next experiments.
- Safety: must distinguish measured metrics from estimates.
- Example: compare hook variants by click-through rate.

### Agent Compliance
- Objective: block risky claims and enforce human approval gates.
- Input: copy, creative, product facts, campaign settings.
- Output: pass/block result, reasons, required edits.
- Safety: blocks invented price, false discount, exaggerated promise, spam, brand misuse, and fake testimonials.
- Example: reject "50% off today" unless verified by a human source.

### Agent Growth
- Objective: propose experiments for organic growth and funnel improvement.
- Input: metrics report, campaign history, creator constraints.
- Output: prioritized experiments and expected learning.
- Safety: cannot launch paid ads, scrape users, or automate spam.
- Example: recommend testing two CTAs against "Comenta TELÃO".
